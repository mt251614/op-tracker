# OP Tracker SPEC.md

## 一、專案目標

製作一個名為 OP Tracker 的前端小程式，作為開盤溢價率計算與短線強弱觀察工具。使用者輸入股票資料與價格後，系統自動計算開盤溢價率，並判斷是否站穩開盤價，最後整合顯示操作建議與風險提醒。

不需要後端與資料庫，歷史紀錄儲存在瀏覽器 `localStorage`。

## 二、輸入欄位

畫面需提供以下欄位：

1. 股票代號
2. 股票名稱
3. 日期
4. 昨日收盤價
5. 今日開盤價
6. 9:05價格
7. 9:10價格
8. 備註

其中「昨日收盤價」與「今日開盤價」為必要欄位。「9:05價格」與「9:10價格」用於站穩開盤價判斷。

## 三、計算公式

開盤溢價率公式：

```text
((今日開盤價 - 昨日收盤價) / 昨日收盤價) * 100
```

結果需以百分比顯示，並保留小數點後 2 位。

## 四、開盤溢價率規則

| 開盤溢價率 | 強弱判斷 | 基礎提醒 |
|---|---|---|
| < 0% | 負溢價，直接撤離 | 開盤弱於昨日收盤價，不列入觀察，不進場。 |
| = 0% | 平盤開出 | 沒有明顯強勢，不列入主要觀察。 |
| > 0% 且 < 3% | 小幅開高 | 強度不足，暫不列入主要觀察。 |
| 3% ~ 5% | 強勢開盤 | 主要觀察區，需搭配是否站穩開盤價判斷。 |
| 5% ~ 7% | 強勢偏熱 | 開盤很強，但追高風險增加，需搭配是否站穩開盤價判斷。 |
| > 7% | 過熱開盤 | 溢價過高，不建議追高，觀察是否開高走低。 |

判斷邊界：

* 3.00% 屬於「強勢開盤」
* 5.00% 屬於「強勢開盤」
* 大於 5.00% 且小於等於 7.00% 屬於「強勢偏熱」
* 大於 7.00% 屬於「過熱開盤」

## 五、站穩開盤價判斷

站穩判斷使用「今日開盤價」、「9:05價格」、「9:10價格」。

| 條件 | 站穩判斷 | 站穩操作建議 |
|---|---|---|
| 9:05價格 >= 今日開盤價 且 9:10價格 >= 今日開盤價 | 站穩開盤價 | 可繼續觀察 |
| 9:05價格 < 今日開盤價 且 9:10價格 < 今日開盤價 | 沒站穩開盤價 | 取消觀察，可能開高走低 |
| 9:05價格 < 今日開盤價 且 9:10價格 >= 今日開盤價 | 轉強觀察 | 可觀察，但不要急追 |
| 9:05價格 >= 今日開盤價 且 9:10價格 < 今日開盤價 | 轉弱 | 小心開高走低 |

若未輸入 9:05價格或 9:10價格，畫面仍可計算開盤溢價率，但站穩判斷顯示「尚未判斷」。

若有輸入 9:05價格或 9:10價格，需檢查輸入內容必須是數字且不可為負數。

## 六、整合操作規則

1. 開盤溢價率 < 0%，顯示「直接撤離」，不需要判斷是否站穩。
2. 開盤溢價率 0% ~ 3%，顯示「不列入主要觀察」。
3. 開盤溢價率 3% ~ 5%，若站穩開盤價，顯示「主要觀察」。
4. 開盤溢價率 3% ~ 5%，若沒站穩開盤價，顯示「取消觀察」。
5. 開盤溢價率 5% ~ 7%，若站穩開盤價，顯示「謹慎觀察」。
6. 開盤溢價率 5% ~ 7%，若沒站穩開盤價，顯示「不追高」。
7. 開盤溢價率 > 7%，顯示「過熱，不追高」。
8. 若溢價率需要站穩資訊但尚未輸入完整 9:05 與 9:10 價格，操作建議先依溢價率顯示「等待站穩判斷」。

整合規則中的「站穩開盤價」包含站穩判斷結果為「站穩開盤價」或「轉強觀察」。

整合規則中的「沒站穩開盤價」包含站穩判斷結果為「沒站穩開盤價」或「轉弱」。

## 七、操作口訣

畫面需顯示：

```text
負數撤離，低於3不看，3到5主觀察，5到7小心追，超過7不追高。
```

## 八、功能需求

1. 使用者可以輸入股票代號、股票名稱、日期、昨日收盤價、今日開盤價、9:05價格、9:10價格與備註。
2. 點擊「計算」後，自動計算開盤溢價率。
3. 若 9:05價格與 9:10價格皆有輸入，需顯示站穩開盤價判斷。
4. 結果區需顯示：
   * 開盤溢價率
   * 強弱判斷
   * 站穩判斷
   * 操作建議
   * 風險提醒
5. 計算成功後，可以點擊「儲存紀錄」。
6. 歷史紀錄使用 `localStorage` 儲存在瀏覽器。
7. 重新整理頁面後，歷史紀錄仍需存在。
8. 歷史紀錄需用表格顯示。
9. 可以刪除單筆紀錄。
10. 可以清空全部紀錄。
11. 可以匯出 CSV。
12. 畫面需支援手機與電腦瀏覽。
13. 程式可直接用瀏覽器開啟 `index.html` 使用。

## 九、錯誤檢查

需檢查以下情況：

1. 昨日收盤價不可為空。
2. 今日開盤價不可為空。
3. 價格輸入內容必須是數字。
4. 昨日收盤價不可為 0。
5. 價格不可為負數。
6. 若輸入錯誤，需顯示錯誤提示，不能計算，也不能儲存錯誤資料。
